# Technical Specification
## Time-Place Celebrity Walkthrough Generator (TPCWG)

## 1. High-Level Pipeline
The video generation process follows these main steps:

1.  **Subject Discovery:** Identify a list of plausible subjects for the given location and time.
2.  **Ranking & Selection:** Score and rank the candidates, then select the top N subjects. An LLM refines this list.
3.  **Image Generation:** For each subject, generate a "hero still" image that places them in the specified venue and era.
4.  **Video Clip Generation:** For each subject image, generate a short video clip (image-to-video) that simulates the camera landing on them.
5.  **Stitching:** Concatenate the individual video clips into a single timeline. This can be a simple crossfade or use dedicated transition clips.
6.  **Labeling:** Add text overlays (title card, subject names, ages) to the stitched video.
7.  **Output:** Emit the final `final.mp4` and a report.

## 2. System Architecture & Dependencies
The system is designed to be run from a single shell script (`run.sh`) and relies on a few key external and local tools:
- **Dependencies:** `bash`, `curl`, `jq`, `ffmpeg`.
- **APIs:** The system is provider-agnostic, configured via `job.json`. It requires endpoints for:
    - An LLM (for subject refinement).
    - An Image Generation model.
    - An Image-to-Video Generation model.
- **Environment Variables:** API keys must be available as environment variables (e.g., `LLM_API_KEY`, `IMAGE_API_KEY`).

## 3. Subject Discovery & Ranking
This is a multi-step process to build the cast list.

### 3.1. Data Sourcing (Default Strategy)
- **Source:** Wikidata via a SPARQL query.
- **Query Logic:** Find people who are associated with the `location` and have an active period overlapping the `date`. The query should filter by occupation (e.g., musician, actor).
- **Output:** A raw JSON list of candidates with their name, QID, birth date, and evidence of plausibility.

### 3.2. LLM Refinement
- **Input:** The raw list of candidates from Wikidata.
- **Task:** The LLM is prompted to:
    1.  Clean the list by removing obvious errors.
    2.  Select the final `num_subjects`.
    3.  Generate `prompt_image` and `prompt_video` text for each subject.
    4.  Output a strictly formatted JSON object matching the `subjects.json` schema.

## 4. Visual Generation
### 4.1. Image Generation
- **Goal:** Create a still frame for each subject.
- **Prompting:** The `prompt_image` generated by the LLM should include details from the `job.json` (`venue`, `date`, `era_style`, `camera_style`) and place the subject within a crowd.

### 4.2. Video Generation
- **Goal:** Create a short clip for each subject.
- **Technique:** Image-to-Video (i2v).
- **Input:** The generated subject image (`images/subjects/<k>.png`).
- **Prompting:** The `prompt_video` should describe the camera motion, e.g., "Handheld walkthrough through crowd, then settle focus on the subject."

## 5. Stitching & Overlays
### 5.1. Stitching Strategy
- **Default:** Use `ffmpeg` to apply a simple crossfade between the generated clips.
- **Alternative:** For more advanced providers, generate dedicated video clips that transition from one subject to the next.

### 5.2. Labeling
- **Tool:** `ffmpeg`'s `drawtext` filter.
- **Process:** After the video is stitched, a final `ffmpeg` pass adds the title card and subject labels (name and age). The timestamps for each label must be calculated based on the duration of the clips.
